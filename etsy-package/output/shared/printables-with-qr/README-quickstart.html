Mini Mission Control

Two-device peer-to-peer video call + D-pad + optional micro:bit bridge
v1.0.0 • Call • Command • Control
Welcome, tiny astronaut 🧑‍🚀✨ Thank you for buying Mini Mission Control! Think of it as FaceTime + Game Controller + micro:bit walkie-talkie: open it on two devices, type the same room code, and you are live — video, audio, a D-pad, text, and text-to-speech, all peer-to-peer through the browser.

First 3 Things To Do

1
Open it on two devices. Load the live URL https://abourdim.github.io/mission-control/ in Chrome on both. A laptop + a phone is the classic combo.
2
Type the same room code, then Start camera + Connect. Both devices type the same code (pizza, alpha-42, whatever). Press 🎥 Start camera, allow permissions, then ✅ Connect.
3
Play. Use the D-pad (UP/DOWN/LEFT/RIGHT/STOP). Type text — press Enter to send. Tap Speak or Remote speak for TTS. Optional: pair a BBC micro:bit V2 over BLE — the D-pad will forward to it.

Key Features

Call: two-device peer-to-peer video + audio (PeerJS) • room-code pairing

Command: D-pad UP/DOWN/LEFT/RIGHT/STOP • text messages • Speak (local TTS) • Remote speak

Control: optional micro:bit V2 BLE UART bridge — D-pad forwards to the board

Polish: camera switch • mic mute • mirror self/remote • remote fullscreen • snapshot PNG • WebM recording • HUD overlay • log panel • cache cleaner

Privacy

Peer-to-Peer — Your Media Doesn't Sit on a Server

Video and audio flow directly between the two devices through WebRTC. A tiny signalling service only helps the two browsers find each other — the call content itself is peer-to-peer.

Keyboard Shortcuts

Only two: Enter to send a text message, Escape to close the firmware modal. Everything else is a button or a tap.
