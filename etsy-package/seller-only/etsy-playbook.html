Full description

🎮 Mini Mission Control — Call • Command • Control. Open the page on two devices. Type the same room code. Press Connect. You're holding a peer-to-peer video call with a D-pad between you — all in the browser, no install, no account. Think: FaceTime + Game Controller + micro:bit walkie-talkie. WHAT YOU GET • 📞 Two-device peer-to-peer video + audio call (PeerJS) • 🔤 Room-code pairing (both devices type the same code) • 🎮 D-pad controller: UP / DOWN / LEFT / RIGHT / STOP sent peer-to-peer • 💬 Text messages over the data channel • 🗣️ Local text-to-speech (Speak) + Remote speak (make the other device talk) • 📡 Optional BBC micro:bit V2 bridge — D-pad forwards to the board over BLE UART • 🔄 Camera switch, 🎙️ mic mute, 🪞 mirror self / remote, ⛶ remote fullscreen • 📸 Snapshot (PNG download), ⏺ recording (WebM download) • 🛰️ HUD overlay, 📋 log panel, 🧹 cache cleaner 5-STEP SETUP 1. Open the page on two devices 2. Type the same room code on both 3. Press 🎥 Start camera (allow permissions) 4. Press ✅ Connect (one becomes host, one guest) 5. Enjoy — D-pad, text, TTS, optional micro:bit link WHAT'S IN YOUR DOWNLOAD • index.html — the full app • js/ — PeerJS wiring, D-pad, TTS, optional micro:bit bridge • docs/ — illustrated guide, quick reference, FAQ, Etsy playbook • Printable extras: quickstart card, A3 classroom poster, lesson plan, stickers • LICENSE — personal / classroom single-user license REQUIREMENTS • Two devices with browsers (Chrome recommended) • Camera + microphone on each device • A live network connection (this is a peer-to-peer call, not offline) • Optional: BBC micro:bit V2 for the BLE bridge • HTTPS required for Bluetooth KEYBOARD SHORTCUTS Only two: Enter (send text), Escape (close firmware modal). WHO IT'S FOR Teachers running STEM / digital-communication workshops · home-schooling families · robotics clubs · makers prototyping P2P projects · anyone curious about peer-to-peer web tech. PRIVACY FIRST Video and audio flow directly between the two devices. A tiny signalling service only helps them find each other — the call content itself stays peer-to-peer. SLUG — AVOID THESE

Common mistakes

❌ Don't✅ Do
Start with "Hi, I'm…" + 10s of introOpen with both devices already on-screen, room code being typed
Film landscape (16:9)9:16 portrait — Etsy autoplays on mobile
Use two devices that are too closeSeparate them enough that mics don't feedback, or mute one
Forget captions (Etsy plays muted)Burn in captions or add a text overlay every scene
Show the micro:bit bridge without HTTPSAlways demo the BLE bridge from the live HTTPS URL
Skip the CTASay "link below" in the last 5 seconds every single time
